# 🧠 Deep Agents Research Console

A multi-agent research assistant built on [`deepagents`](https://github.com/langchain-ai/deepagents),
LangGraph, and Streamlit — extending the original `Deep-agents-With-Langchain`
tutorial project with a production-leaning architecture: a Supervisor /
Researcher / Critic agent team, live progress tracking, SQLite-backed
session history, and Markdown/HTML/PDF export.

> This builds **on top of** the original `deepagentsdemo/` notebooks rather
> than replacing them — the notebooks still work standalone and document the
> core `deepagents` concepts (planning, virtual file system, context
> engineering, backends, subagents) this app assembles into a full product.

---

## ✨ Features

### 1. Multi-agent architecture (Supervisor → Researcher(s) → Critic)
Rather than hand-rolling a second LangGraph graph next to `deepagents`, the
team is built **from deep-agent subagents**, which gives planning and
context isolation for free:

- **Supervisor** (the main agent) — breaks the request into a plan with
  `write_todos`, delegates research questions, and synthesizes the final,
  cited answer.
- **Researcher** subagent(s) — 1 to 3, configurable in the sidebar. Each
  researches a single question in an isolated context window (so a noisy
  10-result search doesn't pollute the supervisor's context), using
  snippets-first search + `think_tool` reflection, and returns a structured
  `ResearchFindings` object (summary, key points, confidence, sources,
  knowledge gaps).
- **Critic** subagent — reviews the supervisor's draft answer against the
  researchers' findings for unsupported claims or missing citations
  *before* the answer reaches the user, returning a structured
  `CritiqueResult` (verdict + flagged claims).

### 2. Production-style Streamlit UI
- Live phase tracking via `st.status` (Planning → Researching → Reviewing →
  Done), updated in real time as the agent's tool calls come in.
- Every step — planning, delegation, search, full-content fetch,
  reflection, file ops — rendered as a labeled, collapsible expander so
  the agent's reasoning is inspectable, not a black box.
- Sidebar **research history**: every session is listed (title, model,
  timestamp), with **Open** (resume the full transcript) and **Delete**
  controls.
- **Export** any finished answer as Markdown, HTML, or PDF with one click.

### 3. Memory & persistence (SQLite)
A lightweight `sqlite3`-backed store (`app/storage/db.py`, no ORM) tracks:
- `sessions` — one row per conversation thread (title, model, backend, status)
- `messages` — the full transcript per session
- `todo_snapshots` — the latest plan state per session

Sessions survive an app restart — reopen the app and your research history
is still in the sidebar.

### 4. Context engineering
- **Snippets-first search**: `internet_search` returns cheap title+snippet
  results for many queries; the separate `fetch_full_content` tool is only
  called on the 1–3 URLs that are actually worth the extra context tokens.
- Agent **Skills** (`/skills/`) and durable **AGENTS.md** context, carried
  over from the original project.
- Bulk content gets written to the virtual file system instead of bloating
  the conversation — also carried over from the original design.

### 5. Agent reflection
- A `think_tool` (`app/tools/reflection.py`) gives the researcher an
  explicit place to reason — "what do I know, what's missing, do I need
  another round" — shown in the UI as a distinct 🤔 **Reflection** step.
- The **Critic** subagent performs citation/claim verification on every
  answer before it's returned.

### 6. Export
- Markdown (`.md`), HTML (`.html`), and PDF (`.pdf`) — all generated by
  `app/export/exporters.py`. PDF uses **reportlab** rather than weasyprint
  or pdfkit specifically because it's pure-Python with no system-level
  dependency (no Pango/Cairo or wkhtmltopdf binary to install), which
  matters for deploying on Streamlit Community Cloud.

## 🔧 Tech stack

| Layer | Choice | Why |
|---|---|---|
| Agent framework | [`deepagents`](https://pypi.org/project/deepagents/) + LangGraph | Planning, virtual FS, subagents, swappable backends out of the box |
| LLM providers | OpenAI, Groq, **OpenRouter** | OpenRouter gives access to free-tier models (e.g. DeepSeek, Llama 3.3) for zero-cost demoing |
| Search | Tavily | Snippets-first search + `.extract()` for full-content fetch |
| UI | Streamlit | `st.status`, `st.chat_message`, `st.download_button` |
| Persistence | SQLite (stdlib `sqlite3`) | Zero extra infra; good enough for session history |
| PDF export | reportlab | Pure-Python, no system dependency — deploys cleanly |

## 🚀 Setup

```bash
# 1. Clone / unzip the project, then from its root:
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Configure API keys
cp .env.example .env
# then edit .env and fill in the keys for the provider(s) you want to use
```

You need:
- **`TAVILY_API_KEY`** — required for any research feature ([tavily.com](https://tavily.com), free tier available)
- At least one of:
  - **`OPENAI_API_KEY`**
  - **`GROQ_API_KEY`**
  - **`OPENROUTER_API_KEY`** — get a free key at [openrouter.ai](https://openrouter.ai); several models on the
    model dropdown (suffixed `:free`) cost nothing to call

```bash
# 4. Run
streamlit run streamlit_app.py
```

The app opens at `http://localhost:8501`.

## ☁️ Deployment — Streamlit Community Cloud

1. Push this repo to GitHub (make sure `.env` is **not** committed — it's
   already in `.gitignore`).
2. Go to [share.streamlit.io](https://share.streamlit.io) → **New app**.
3. Pick your repo/branch and set **Main file path** to `streamlit_app.py`.
4. Under **Advanced settings → Secrets**, add your keys in TOML format:
   ```toml
   OPENAI_API_KEY = "sk-..."
   TAVILY_API_KEY = "tvly-..."
   OPENROUTER_API_KEY = "sk-or-..."
   ```
   Streamlit Cloud injects these as environment variables, so
   `load_dotenv()` + `os.getenv(...)` in this app picks them up the same
   way it would from a local `.env` file.
5. Deploy. First boot installs `requirements.txt`, which can take a couple
   of minutes the first time (mainly `langchain`/`deepagents` and friends).

**Note on persistence:** Streamlit Cloud's filesystem is ephemeral across
redeploys/restarts — the SQLite file under `data/` will reset when the app
restarts or sleeps from inactivity. For durable history across redeploys in
a real production deployment, swap `app/storage/db.py`'s SQLite connection
for a hosted database (e.g. Postgres/Supabase) — the function signatures
in that module are deliberately small and self-contained, so swapping the
implementation doesn't require touching `streamlit_app.py`.

## 🗺️ Possible next steps

- True parallel researcher execution (currently the supervisor delegates
  sequentially via `task`; LangGraph supports fan-out/fan-in if you want
  genuinely concurrent subagent runs).
- Streaming token-by-token output instead of `agent.invoke()` (deepagents/
  LangGraph support `.stream()` — would pair well with the live phase
  indicator).
- Swap SQLite for Postgres for durable history on platforms with ephemeral
  disks.
- Email export — left out of this build because reliable transactional
  email needs a real provider (SES/SendGrid/etc.) and per-deployment
  config; the Markdown/HTML exports are a clean substitute (attach the
  download to an email manually, or wire in a provider's API to
  `app/export/exporters.py` output if you want this automated).
